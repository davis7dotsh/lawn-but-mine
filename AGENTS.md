We have a few "philosophies" I want to make sure we honor throughout development:

### 1. Performance above all else

When in doubt, do the thing that makes the app feel the fastest to use.

This includes things like

- Optimistic updates
- Using the custom data loader patterns and custom link components with prewarm on hover
- Avoiding waterfalls in anything from js to file fetching

### 2. Good defaults

Users should expect things to behave well by default. Less config is best.

### 3. Convenience

We should not compromise on simplicity and good ux. We want to be pleasant to use with as little friction as possible. This means things like:

- All links are "share" links by default
- Getting from homepage to latest video should always be fewer than 4 clicks
- Minimize blocking states to let users get into app asap

### 4. Security

We want to make things convenient, but we don't want to be insecure. Be thoughtful about how things are implemented. Check team status and user status before committing changes. Be VERY thoughtful about endpoints exposed "publicly". Use auth and auth checks where they make sense to.

## Cursor Cloud specific instructions

### Overview

Lawn is a single-package video review app (not a monorepo). Tech stack: React 19 + TanStack Router/Start, Convex (local backend in dev), Clerk auth, Mux video, Tailwind CSS 4, Vite 7. Package manager is **Bun**.

### Running the app

- `bun run dev` — starts Vite (port 5296) **and** a local Convex backend (port 3210) via `convex-vite-plugin`. The `USE_LOCAL_CONVEX=true` flag is set automatically by this script.
- The Convex local backend binary is downloaded on first run to `~/.convex-local-backend/` and cached for subsequent runs.
- A `.env` file must exist at the project root with at minimum `VITE_CLERK_PUBLISHABLE_KEY` (or `CLERK_JWT_ISSUER_DOMAIN`) and `AUTUMN_SECRET_KEY`. Without these, `bun run dev` will throw at startup. All secrets are injected as environment variables; the `.env` file is generated from them.

### Quality checks

- `bun run lint` — ESLint across `app/`, `src/`, `convex/` (ignores `convex/_generated/`)
- `bun run typecheck` — TypeScript check for the app
- `bun run typecheck:convex` — TypeScript check for Convex functions
- `bun run check` — runs lint + both typechecks

### Gotchas

- The `.env` file is not committed. If secrets are available as environment variables, generate `.env` from them before starting the dev server.
- `convex/_generated/` is auto-generated by the Convex deploy step during `bun run dev`. Do not edit these files.
- The Vite config (`vite.config.ts`) validates required env vars at startup when `USE_LOCAL_CONVEX=true`. Missing vars produce a clear error message.
- `bun run dev:web` starts only the Vite server (no Convex backend) — useful for frontend-only work, but Convex queries will fail without a backend.
- No Docker, no containers, no Makefile. All infrastructure is either local (Convex dev backend) or external SaaS (Clerk, Mux, Railway S3, Autumn).
